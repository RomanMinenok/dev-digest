Вендорний `Dropdown` (`vendor/ui/kit/Dropdown.tsx`) рендерить **лише** плаский

*Твоє рішення вранці:* лишити як є, чи розширити вендорний kit окремою таскою.

### D-9 · T-25: статус лейна береться з persisted `status`, а не напряму з SSE
`useRunEvents` (`lib/hooks/reviews.ts:183-227`) віддає **агрегований** булеан
`running` на весь набір ранів — а не по кожному окремо. Правити хук було поза
скоупом T-25.
**Рішення:** статус кожного лейна деривується з власного persisted `status`
учасника (fail-closed, дзеркалить серверний `deriveStatus`), а SSE-потік дає
(а) живий рядок останньої події в шапці лейна поки run триває, (б) одноразовий
`refetch` коли всі потоки закрились — той самий патерн, що вже вживає `RunStatus.tsx`.
AC-24 («живий статус, без полінгу») виконано: полінгу немає, `refetchInterval` — нуль.
*Якщо вранці схочеш «правильніше»* — треба розширити `useRunEvents` до
статусу per-run. Це окрема таска, вночі без тебе не ліз.
